## 어떻게 디자인을 시작해야 할까요?

디자인의 기초는 많이 보고 따라하기 라고 생각합니다.

pinterest나 behance 등 웹서비스 & 어플리케이션 관련된 레퍼런스가 많으니 일단 먼저 접해보는것이 중요합니다.

제가 좋아하는 디자인 사이트인 “토스”에서는 다음과 같은 디자인 원칙을 준수하고 있습니다.

1. 제이콥의 법칙 : 사용자는 익숙한 환경을 원한다
    - 즉, 사용자가 새로운 어플리케이션이나 웹서비스를 접할때 설명서를 읽고싶지 않아합니다.
    - 최대한 자신이 기존에 써왔던 로직을 가져오고 싶어합니다.
    
    → 예시 : 네이버 vs 구글
    
    → 예시 : 어떤 아이콘들이 보통 어떤 역할을 하는지 정해져 있습니다.(토글버튼,로그인버튼,장바구니 버튼 등)
    
2. 힉의 법칙 : 사용자는 선택지가 많으면 혼란스러워한다
    - 선택지가 너무 많으면 사용자가 이를 자세히 읽으려는 시도를 선택하는 대신, 그 서비스를 계속 사용하지 않습니다.
    - 왜냐하면 결정에 걸리는 시간이 너무 길어지기 때문입니다.

## 개발자의 시선에서 바라본 UX UI 디자인의 중요성

앞서 언급한 UX,UI 법칙을 준수하지 않은채, 어 이거 예쁜데? 이거 괜찮은데? 라고 특별한 의도 없이 추가하는 순간 서비스를 사용하는 **사용자는 혼란스러움**을 느끼게 됩니다.

- “항대생활<<KAU광장”에 숨겨져 있는 공지사항
- “항대생활” 카테고리를 클릭하지 않으면 펼쳐진 부분(아코디언이라고 부릅니다)가 없어져버림

**사용자의 행동을 통제할수 없게 됩니다.**

- 여기 강의실을 빌리고 싶다고 칩시다.
- 놀랍게도 종정시 말고 다른 곳에서도 강의실을 빌릴수가 있어요. 어디있을까요?

**비효율적이고 혼란스러운 인터페이스는 사용자의 혼란을 가중시킵니다.**

- 전체적으로 모두 비효율적이긴 합니다만,
- DDC가 뭔가요
- 저희학교 취업률이 0%인가요?

저희는 어쩔수 없이 우리학교 홈페이지를 계속 써야하지만, 우리학교 아닌 학생입장에서 이 홈페이지를 계속 쓸 이유가 있을까요?

## 피그마 기본

`프레임이 뭔가요?`

- 프레임은 디자인의 기본 단위입니다.
    - 박스, 즉 컨테이너 역할을 하고 있다 보시면 됩니다.
    - 특히 웹/모바일에서 모두 적용되는 페이지를 반응형이라고 부르는데 각각 디자인 할수 있습니다.
    - 반응형은 Constraint(제약조건)을 설정함으로서 세부적인 디자인을 조정할수 있습니다.
    - 예시로 presentation, Desktop, mobile 에서 선택할수 있습니다.

## 신제품 페이지를 만들어봅시다

1. 전체 신제품 페이지를 위한 프레임을 생성합니다. → 가로크기를 1920px로 시작하겠습니다.
2. 프레임 이름을 기본으로 두지말고 Newpage로 설정하여 협업에 용이하게 합시다.
3. 회색배경의 디퓨저 사진을 1920*1050 크기로 만들어줍시다

### 헤더 만들기

1. 네모를 그려 볼까요? 세로 크기는 120
2. 배경을 흰색으로 채울거고 투명도는 25%로 줍니다.
3. T를 눌러서 New를 입력해주고 폰트는 `kaisei Decol`로 설정합니다
- 이때 alt + shift 키를 누르면 수전증이 있는 사람도 쉽게 복붙이 가능하다
    - 대충 4개로 복사 붙여넣기 하는 과정에서, 이때 사이 간격이 픽셀로 주어집니다
    - 정렬을 하고싶으면 position 탭에서 정렬을하면 됩니다

### Autolayer 생성하기

주로 담당하는 기능은 자동정렬입니다.

- 다른 AutoLayer간의 자동정렬을 해줍니다.
- AutoLayer를 구성하는 요소들 간의 자동정렬을 해줍니다.
- 버튼 : 텍스트 길이와 크기에 따라 자동 조절이 가능해집니다.
- 네비게이션바 : 메뉴들이 수평으로 조정되고 크기를 변경할 수 있습니다.

`단축키 : shift + A`

![image.png](attachment:2d414780-c25d-4399-b8d6-7a66c1d635c0:image.png)

### 푸터 만들기

1. 네모를 만듭니다. → 색상코드는 #E7EFF5, 높이는 380
2. 색깔 코드를 일일이 입력하기 힘드니, 색상의 변수는 이렇게 정할수 있습니다.

![image.png](attachment:b28eea87-5ac1-40a2-8b2b-ebe021e6f5b5:image.png)

- 푸터 안에 있는 내용들을 만들고 각각 그룹화를 한다 → 그룹화의 경우 구성 내용들을 바꾸는데 크기가 거슬리지는 않는데
- #로 프레임화하고 싶다면 alt+ctr+G로 하면 된다(혹은 우클릭,프레임화)
    - 예시에서는 이렇게 운영진들이 만들어주셨으니 이걸 따라서 만들어봅시다.
    - `주의사항` : 만든 프레임의 크기가 두개의 텍스트를 합친만큼의 크기랑 똑같아야함

![image.png](attachment:80a92d52-a9d9-4430-b8a1-021bee522c3b:image.png)

푸터 각 요소를 space-between 정렬하려면 저기 정렬부분을 더블클릭하면 됩니다.
